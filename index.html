02

3D-printed parts

Two printed parts are needed. Both are available for free on MakerWorld for private, non-commercial use.

Part 1 — Socket adapter (Low-Profile Plus → Bitaxe)

Mounts the 52Pi heatsink on the PCB mounting points. The MakerWorld version is much sturdier than the original adapter — you can tighten the included screws without risking breakage.

Material: Print in PETG or ASA for better heat resistance. PLA can warp in a warm environment. 0.2mm resolution, at least 3 perimeters.

Part 2 — 60mm fan shroud

The heatsink ships with a noisy, non-standard 50×50mm fan (3-pin micro connector). The shroud lets you mount a Noctua NF-A6x15 60mm fan on the heatsink's original mounting brackets. The 50→60mm reduction compresses airflow and noticeably improves cooling efficiency.

Pull vs Push airflow: Testing shows that in pull mode (fan pulling air through the heatsink rather than pushing), ASIC temperature drops an additional 3.5°C at 1,000 MHz. Test both orientations after running the benchmark.

Copper shims (recommended)

The heatsink base measures 30×22mm but the BM1370 die is smaller. Placing two thin copper shims (0.3mm) on either side of the ASIC contact point distributes heat across all 4 heatpipes instead of just 2. Apply thermal paste on all contact surfaces.

03

Cooling assembly

Assembly requires precision and clean thermal surfaces. Follow these steps in order with the Bitaxe powered off.

  1. Clean the ASIC — Remove any stock heatsink if present. Clean the BM1370 die surface with isopropyl alcohol (90%+) and a cotton swab. Let dry for 60 seconds.

  2. Prepare copper shims (optional) — Place the two 0.3mm shims on either side of the heatsink base center, where the ASIC contact will be. Apply a thin layer of thermal paste between the shims and the heatsink base.

  3. Apply thermal paste — Place a small amount of Thermal Grizzly Kryonaut Extreme at the center of the die. Spread evenly across the full ASIC surface and over the copper shim contact zones.

  4. Mount the socket adapter — Screw the 3D-printed adapter onto the 4 PCB mounting points. Tighten in a cross pattern progressively — do not fully tighten one side before the others.

  5. Install the heatsink — Set the 52Pi Low-Profile Plus onto the adapter. Align the screw holes and fasten with the included screws. Cross-pattern tightening again. Verify firm, even contact.

  6. Mount shroud + fan — Attach the 3D-printed shroud to the heatsink brackets, then clip the Noctua NF-A6x15 into the shroud. Plug the 4-pin PWM connector into the Bitaxe fan header.

🚨 Never power on without a properly mounted heatsink. The BM1370 ASIC can be permanently damaged in seconds without cooling.

04

PSU wiring

The Meanwell LRS-100-5 (5V / 20A) can power up to 2 Bitaxe Gamma simultaneously, even when heavily overclocked (~20–25W per unit).

🚨 DANGER — Mains voltage: The AC input terminals are at lethal voltage. Never work on the AC wiring without unplugging the unit from the wall first.

AC side (mains input — 120/240V)

L (Live)→Brown/black wire from mains cable → L terminal on Meanwell
N (Neutral)→Blue/white wire from mains cable → N terminal on Meanwell
⏚ Earth→Green/yellow wire → Earth terminal on Meanwell. Always connect.

DC side (5V output to Bitaxe)

Reuse the cables included with each Bitaxe. Strip the ends cleanly to about 8mm and connect to the DC output terminals.

+V (5V)→Red/orange Bitaxe cable wire → +V terminal on Meanwell
-V (GND)→Black Bitaxe cable wire → -V terminal on Meanwell

Output voltage adjustment

The Meanwell has a fine-adjustment trimmer potentiometer. Using a multimeter, measure the DC output voltage and adjust to exactly 5.00V (tolerance: 4.95–5.05V) before connecting the Bitaxe units.

ℹ Capacity note: The LRS-100-5 delivers up to 20A continuous. Two Bitaxe at 1,000 MHz draw ~80W total (~16A) — well within the safe operating range with a good margin.

05

Initial AxeOS setup

Before launching the automated benchmark, configure the Bitaxe to safe starting values via the AxeOS web interface (access it by typing the miner's IP address in your browser).

⚠ Recommended starting values: Begin at 500 MHz / 1,150 mV — this is the benchmark script's default starting point. Never start directly in extreme territory.

06

Python benchmark script

The Bitaxe-Hashrate-Benchmark script (by thomasKn) fully automates finding the best frequency/voltage pair for your specific ASIC. It tests progressive steps, monitors temperature in real time, and automatically applies the best stable settings found at the end.

Installation

Prerequisites: Python 3.11 or higher, and your Bitaxe reachable on the local network.

# 1. Clone the repository git clone https://github.com/thomasKn/Bitaxe-Hashrate-Benchmark.git cd Bitaxe-Hashrate-Benchmark # 2. Create and activate a virtual environment python -m venv venv # On macOS / Linux: source venv/bin/activate # On Windows: venv\Scripts\activate # 3. Install dependencies pip install -r requirements.txt

Usage

Run the script with your Bitaxe's IP address. You can optionally specify a starting voltage and frequency.

# Basic command (defaults: 500 MHz / 1,150 mV) python bitaxe_hashrate_benchmark.py 192.168.x.xxx # With custom starting parameters python bitaxe_hashrate_benchmark.py 192.168.x.xxx -v 1175 -f 600 # Available options: # -v, --voltage Starting voltage in mV (default: 1150) # -f, --frequency Starting frequency in MHz (default: 500)

ℹ Finding your Bitaxe's IP: Log into your router's admin panel and look for a device named "bitaxe" or "esp32". Alternatively, run nmap -sn 192.168.x.0/24 in a terminal to scan your network.

How the script works

Init
System info fetch — The script reads the current Bitaxe configuration via the AxeOS API (/api/system/info, /api/system/asic): core count, current voltage, current frequency.
Step
Apply settings and stabilize — The script pushes the voltage/frequency pair via PATCH /api/system, restarts the Bitaxe, then waits 90 seconds for the hashrate to stabilize.
Measure
10-minute benchmark — Samples every 15 seconds: hashrate, ASIC temperature, VReg temperature, input voltage, power draw in watts, error percentage. The 3 highest and 3 lowest hashrate readings are discarded to eliminate outliers.
Decide
Progression logic — If hashrate is within 94% of the theoretical maximum: increase frequency by +25 MHz. If hashrate is too low: step frequency back down and increase voltage by +20 mV. Repeat until thermal or stability limits are hit.
Done
Apply best settings — The script automatically applies the highest stable hashrate configuration, saves all results to a timestamped JSON file, and prints a top-5 hashrate + top-5 efficiency summary.

⚠ Total duration: Each step takes ~12 minutes (90s stabilization + 10min measurement). A full benchmark from 400 MHz to 1,000+ MHz can take several hours. Plan accordingly and do not cut power during the process.

08

Quad cooling tower (optional)

Want to stack 4 Bitaxe Gamma in a compact build with a single 120mm fan? The 3D-printed quad cooling tower makes it possible.

What you need for the ×4 tower

• 4× Bitaxe Gamma (each with its 52Pi Low-Profile Plus + Noctua shroud)

• 1× 3D-printed tower — MakerWorld

• 1× 120mm fan (Noctua NF-F12 or NF-P12 recommended)

• 2× Meanwell LRS-100-5 (one per pair of overclocked Bitaxe)

ℹ Power for 4 Bitaxe: 4 Bitaxe at 1,000 MHz = ~160W total. A single LRS-100-5 (100W max) is not enough. Plan for 2× LRS-100-5, or upgrade to a LRS-200-5 (5V/40A).

09

Tips & troubleshooting

🌡️

Target temperatures

ASIC: aim for <60°C at max load. VReg: stay under 85°C. The script auto-cuts at 66°C ASIC / 86°C VReg.

📊

HW Errors in AxeOS

An error rate >2% means instability. The script handles this automatically by bumping voltage +20 mV.

🧵

Print material

PETG or ASA for parts near the heatsink (rated to ~80–100°C). PLA is not suitable — it will warp.

🔁

Renew thermal paste

Every 12–18 months, or if you notice temperature drift at constant settings. Always clean thoroughly before reapplying.

⚡

Clean wiring

Use 16 AWG for DC cables. Undersized wire causes voltage drop that destabilizes the miner under load.

🔄

Re-run the benchmark

Re-run if ambient temperature changes significantly (seasons change, new room) or if you modify the cooling setup.
